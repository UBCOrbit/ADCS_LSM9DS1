11 ## Note
12 This project is configured to use mutexes/ semaphores.
13 Your version of HALCoGen might have a bug with generating FreeRTOS code that enables mutexes.
14 
15 This bug is documented [here](http://e2e.ti.com/support/microcontrollers/hercules/f/312/p/626490/2320355).
16 
17 The workaround is to copy the `mpu_wrappers.c` file in `HALCoGen_fix` to `<Your HALCoGen Install>/drivers/FreeRTOS/portable/CCS/Coretex-R4/`.
18 You may want to rename the original file to something like `mpu_wrappers.c.old` just in case you want to restore it.
